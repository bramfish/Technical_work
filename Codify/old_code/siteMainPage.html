Software Engineering Community of Practice

Domain Driven Design

Domain-driven design (DDD) is an approach to software development that prioritizes the business domain of the software over the technical details. It is a way of thinking about software that emphasizes the importance of understanding the business problem that the software is trying to solve, and designing the software in a way that reflects that understanding.

Clean Coding

Clean coding is a set of principles and practices that aim to make code more readable, maintainable, and easier to understand. It emphasizes the importance of writing code that is easy to read and understand, and that is organized in a way that makes it easy to modify and extend.

CQRS Pattern

Command Query Responsibility Segregation (CQRS) is a pattern that separates the read and write operations of a system into separate models. It is a way of designing software that separates the concerns of reading data from the concerns of writing data, which can lead to a more scalable and maintainable system.
